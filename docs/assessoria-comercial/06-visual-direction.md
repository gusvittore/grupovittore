Direção Visual da Landing Page — Grupo Vittore

Este documento define a direção visual final da landing page do Grupo Vittore.

A partir desta versão, a direção visual principal da página deve seguir fielmente as referências visuais da pasta:

public/references/3

Essas referências são a fonte principal de verdade para:

layout;
composição;
fundo;
cores;
tipografia;
espaçamento;
hierarquia visual;
proporção dos títulos;
estilo dos cards;
estilo dos botões;
uso de imagens;
uso de dashboards;
uso de gráficos;
uso de elementos decorativos;
alternância entre seções claras e escuras.

A landing page não deve seguir a regra antiga de ser inteira off-white.

A landing page deve respeitar o visual de cada referência: algumas seções são escuras, outras são claras.

A página também passa a ter uma nova seção de Inteligência, Tecnologia e Automação, posicionada depois da seção Assessoria Personalizada e antes da seção FAQ.

1. Objetivo visual da página

A landing page precisa transmitir:

autoridade;
sofisticação;
estratégia;
inteligência comercial;
crescimento empresarial;
método;
controle;
previsibilidade;
performance;
exclusividade;
confiança;
tecnologia aplicada ao crescimento;
inteligência artificial com propósito comercial.

O visual deve posicionar o Grupo Vittore como uma assessoria comercial e empresarial premium, que une:

marketing;
vendas;
tecnologia;
inteligência;
processo;
dados;
automação;
estratégia.

A página não deve parecer uma agência comum de marketing.

A página não deve parecer um template genérico.

A página não deve parecer uma landing page improvisada.

A página deve parecer uma apresentação visual de alto padrão, com estética forte, institucional e estratégica.

2. Regra visual principal

A regra principal é:

Cada seção deve seguir 100% sua referência visual correspondente dentro da pasta public/references/3.

O Codex deve usar as referências como direção obrigatória, e não apenas como inspiração leve.

Isso significa copiar o máximo possível:

estrutura;
proporção;
posicionamento;
tamanho relativo dos elementos;
direção dos textos;
estilo do fundo;
estilo dos títulos;
ritmo visual;
cards;
imagens;
botões;
elementos decorativos;
respiro;
alinhamento;
composição.

Não criar uma versão própria da seção.

Não misturar o visual antigo com o novo.

Não ignorar o fundo da referência.

Não trocar uma seção escura por clara.

Não trocar uma seção clara por escura.

Não inserir grid nas seções claras se a versão final pedir fundo off-white limpo.

4. Regra de prioridade visual

Quando houver conflito entre este documento, arquivos antigos ou qualquer orientação anterior, a prioridade deve ser:

Referência visual específica da seção em public/references/3;
Estrutura definida em docs/04-page-structure.md;
Copy definida em docs/03-copy-final.md;
Assets finais em public/brand/3;
Regras gerais deste documento.

Se uma referência mostra fundo escuro, usar fundo escuro.

Se uma referência mostra fundo off-white, usar fundo off-white.

Se uma referência mostra layout em duas colunas, usar layout em duas colunas.

Se uma referência mostra título centralizado, centralizar.

Se uma referência mostra título alinhado à esquerda, alinhar à esquerda.

Não aplicar regra geral por cima da referência específica.

6. Fundo e textura
6.1. Fundo escuro

Nas seções escuras, o fundo deve seguir a referência.

O fundo pode conter:

azul marinho profundo;
textura técnica sutil;
linhas comerciais;
gráficos;
elementos de diagnóstico;
elementos de funil;
pontos conectados;
detalhes em dourado;
imagem de impacto;
overlays escuros.

Não usar fundo escuro liso demais se a referência tiver elementos visuais.

Não remover os elementos de fundo se eles fazem parte da referência.

Não clarear a seção se a referência for escura.

6.2. Fundo claro

Nas seções claras, o fundo deve usar o off-white:

#FBF8F4

O fundo deve ser limpo, premium e sofisticado.

A regra atual é:

não usar grid de quadrados nas seções claras, salvo se uma referência específica exigir um elemento decorativo pontual.

Nas seções claras, evitar:

grid pesado;
fundo branco puro;
fundo cinza comum;
excesso de textura;
excesso de elementos técnicos;
poluição visual.

Pode usar elementos gráficos muito sutis apenas quando fizer sentido com a referência.

O fundo claro deve parecer parte da identidade premium da página, não uma área branca genérica.

7. Tipografia

A tipografia é um dos pontos mais importantes da identidade visual.

A página deve usar uma composição tipográfica premium, próxima das referências.

7.1. Títulos principais

Os títulos principais devem usar fonte serifada premium.

Fontes recomendadas:

Cormorant Garamond;
Playfair Display;
Libre Baskerville;
Bodoni Moda.

A fonte deve parecer:

editorial;
sofisticada;
institucional;
premium;
elegante;
com boa presença.

Não usar fonte sans-serif genérica nos títulos principais.

Não usar fonte pesada demais.

Não usar fonte com aparência de template comum.

Não deixar os títulos grotescamente grandes.

Os tamanhos devem seguir a proporção das referências.

7.2. Títulos em seções escuras

Em seções escuras, os títulos podem usar:

branco/off-white;
dourado em palavras específicas;
serifada premium;
peso elegante;
line-height refinado.

A composição deve seguir exatamente a referência da seção.

7.3. Títulos em seções claras

Em seções claras, os títulos devem usar:

azul marinho #000717 como base;
vermelho imperial #960000 para trechos estratégicos;
serifada premium;
line-height elegante;
diagramação parecida com a referência;
peso um pouco mais visível que a versão inicial;
tamanho suficiente para ter presença sem ficar exagerado.

O vermelho deve ser usado apenas nos trechos importantes da mensagem.

Não usar vermelho em excesso.

Não usar dourado como cor principal de headline em seções claras, a menos que a referência mostre isso.

7.4. Eyebrows

Os eyebrows são os textos pequenos acima das headlines.

Eles devem seguir a referência de cada seção.

Características gerais:

uppercase;
letter-spacing amplo;
cor dourada;
fonte sans-serif;
peso médio;
aparência técnica e premium.

Exemplos:

ASSESSORIA DE VENDAS E MARKETING DE PERFORMANCE;
DIAGNÓSTICO DE ESTRUTURAÇÃO COMERCIAL;
PROCESSO COMERCIAL;
MÉTODO E RECEITA;
COMO TRABALHAMOS;
CLIENTES E PARCEIROS;
ASSESSORIA DE VENDAS;
ASSESSORIA PERSONALIZADA;
TECNOLOGIA QUE GERA RESULTADOS;
FAQ.

O tamanho e alinhamento do eyebrow devem seguir a referência da seção.

7.5. Parágrafos

Os parágrafos devem ser legíveis, elegantes e bem espaçados.

Usar fonte limpa e legível.

Características:

cor de acordo com o fundo;
line-height confortável;
largura máxima controlada;
tamanho proporcional à referência;
sem blocos de texto apertados;
sem textos pequenos demais.

Em seções escuras, usar texto claro.

Em seções claras, usar azul marinho.

As descrições dos cards, dashboards, etapas e blocos de conteúdo devem ter tamanho confortável de leitura.

Não usar fonte de formiguinha.

9. Botões

Os botões principais devem seguir o estilo das referências.

Botão verde principal

Características:

fundo verde #008723;
texto branco/off-white;
uppercase;
letter-spacing discreto;
altura confortável;
largura semelhante à referência;
bordas levemente arredondadas;
sombra sutil, se a referência mostrar.

Usar para:

Hero;
Formulário;
CTAs principais.
Regras atuais dos botões

Não usar ícone dentro do botão verde da Hero, se a versão final pedir apenas texto.

Não usar ícone dentro do botão verde do formulário, se a versão final pedir apenas texto.

Não usar linha dourada embaixo do botão verde do formulário.

O botão verde deve parecer limpo, direto e forte.

Botões secundários

Se a referência mostrar botão secundário com borda dourada, seguir o mesmo padrão.

Características:

fundo transparente ou escuro;
borda dourada;
texto dourado ou claro;
aparência premium.

Não criar botões novos que não aparecem na referência.

Características:

fundo azul profundo ou transparente;
borda fina dourada;
textos claros;
ícones dourados;
números em dourado;
composição técnica;
aparência premium.

12.9. Inteligência, Tecnologia e Automação

Esta é uma seção nova da página.

Ela deve entrar depois de:

Assessoria Personalizada

E antes de:

FAQ

Objetivo visual

A seção deve mostrar que o Grupo Vittore também atua com tecnologia, inteligência artificial e automação aplicada ao crescimento comercial.

Ela deve reforçar:

curadoria tecnológica;
implementação prática;
automação de processos;
uso de IA com objetivo de eficiência;
análise de dados;
leitura de performance;
redução de desperdício;
decisões mais lucrativas.

A seção não deve parecer uma seção genérica de tecnologia.

Ela precisa parecer parte da identidade do Grupo Vittore: estratégica, premium, analítica e alinhada ao arquétipo do sábio.

Base visual

A seção deve usar base clara premium:

fundo off-white #FBF8F4;
textos em azul marinho #000717;
destaques em vermelho #960000;
detalhes em dourado #B29157.

Não usar fundo escuro nessa seção, salvo se uma referência futura pedir explicitamente.

Não usar grid de quadrados no fundo.

Composição recomendada

No desktop, usar composição em duas colunas.

Coluna esquerda

A coluna esquerda deve conter:

Eyebrow;
Título principal;
Subtítulo/parágrafo de apoio;
Subtítulo interno “Aplicações estratégicas”;
Três descrições estratégicas;
Marcadores, linha vertical, detalhes dourados ou pontos vermelhos, se fizer sentido visualmente.
Coluna direita

A coluna direita deve conter uma composição visual relacionada a:

inteligência artificial;
automação;
tecnologia;
integrações;
dados;
fluxos;
sistemas conectados.

Pode usar asset pronto da pasta:

public/brand/3

Caso não exista asset pronto, recriar com HTML, CSS ou SVG uma composição premium de linhas, nós, círculos e elementos conectados.

Não usar imagem externa.

Não baixar imagem da internet.

Não usar logos reais de ferramentas se eles não estiverem fornecidos nos assets do projeto.

Título

Título oficial:

“Inteligência, Tecnologia e Automação”

A quebra recomendada:

Linha 1: Inteligência, Tecnologia
Linha 2: e Automação

O trecho “e Automação” pode ficar em vermelho #960000.

“Mapeamos oportunidades de automação e inteligência artificial para simplificar processos operacionais, reduzir tarefas manuais e diminuir custos invisíveis na rotina da empresa.”
“Aplicamos tecnologia nas ações de marketing e vendas para gerar dados mais claros, melhorar a leitura da performance e apoiar decisões com mais precisão.”
“Acompanhamos os indicadores continuamente para entender o que está funcionando, corrigir gargalos e aumentar a conversão com mais controle sobre margens, esforço e resultado.”

Palavras importantes podem receber destaque em vermelho, com moderação:

reduzir tarefas manuais;
diminuir custos invisíveis;
melhorar a leitura da performance;
aumentar a conversão.
Estética

A seção deve parecer:

inteligente;
limpa;
sofisticada;
tecnológica sem parecer futurista genérica;
conectada ao restante da página;
com visual de método e decisão.

Não usar visual muito colorido.

Não usar ícones demais.

Não usar visual infantil.

Não usar estilo SaaS genérico.

13. Espaçamento e proporção

A página deve ter bastante respiro.

O espaçamento deve seguir as referências.

Não deixar seções apertadas.

Não deixar títulos esmagados.

Não deixar cards colados.

Não deixar imagens pequenas demais se a referência usa imagem grande.

Não deixar títulos maiores do que a referência.

Não encolher demais os textos dos cards.

Cada seção deve ter altura e ritmo semelhantes aos prints de referência.

A seção Como Trabalhamos deve ter altura suficiente para respirar, sem parecer compactada.

As seções claras devem manter bastante espaço negativo para preservar o visual premium.

14. Responsividade

A versão desktop deve ser a mais fiel possível às referências.

A versão mobile deve adaptar a composição mantendo o padrão visual.

No mobile:

seções em duas colunas devem empilhar;
títulos devem reduzir sem perder presença;
imagens devem ser reposicionadas sem cortar mal;
cards devem ficar legíveis;
botões devem ter largura confortável;
carrosséis devem permitir scroll horizontal;
FAQ deve funcionar com accordion;
rodapé deve empilhar;
elementos visuais decorativos podem ser reduzidos para evitar poluição;
a seção de tecnologia deve empilhar texto e visual de forma limpa.

Não deixar texto estourar.

Não deixar botão quebrar de forma feia.

Não deixar cards apertados.

Não cortar imagem importante.

Não deixar gráficos ilegíveis no mobile.

15. O que não fazer

Não fazer:

não usar a direção visual antiga como regra principal;
não forçar todas as seções a serem off-white;
não proibir fundo escuro quando a referência for escura;
não transformar a Hero em seção clara;
não remover imagem de autoridade da Hero;
não remover dashboards, gráficos ou elementos visuais quando a referência mostra;
não centralizar tudo se a referência usa composição lateral;
não criar uma versão própria da página;
não simplificar demais;
não misturar cards claros em seções escuras;
não usar fontes genéricas nos títulos;
não deixar headlines gigantes fora da proporção da referência;
não usar cores aleatórias;
não adicionar elementos que não existem na referência;
não usar rodapé escuro se a referência do rodapé é clara;
não usar grid de quadrados nas seções claras quando a versão final pede fundo limpo;
não colocar a seção de Inteligência, Tecnologia e Automação fora da ordem definida;
não usar imagem externa sem autorização;
não usar logos reais de ferramentas se eles não estiverem nos assets do projeto;
não transformar a seção de tecnologia em uma seção SaaS genérica.
